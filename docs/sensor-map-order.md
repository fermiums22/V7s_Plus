# Sensor Map Order

Простой рабочий список. Идем сверху вниз, после прозвонки меняем статус.

## Статусы

- `todo` - еще не прозванивали.
- `probing` - сейчас прозваниваем.
- `mapped` - разъем/узел понятен, пины записаны.
- `firmware` - подключено в прошивке.
- `skip` - не трогаем сейчас.

2. `firmware` - правое колесо `J14`: мотор, энкодер, драйвер.

## Прозваниваем дальше

1. `probing` - `J7`: передняя IR-панель на весь бампер, 8 пинов.
   - На плате элементы подписаны `Dxx` и `Qxx` в шахматном порядке.
   - Крайний правый `D11` был заклеен вспененным скотчем. Запомнить как возможную штатную оптическую маску.
   - Найдено: `J7-3 = GND`, общий полигон одного контакта всех `Qxx`.
   - Обозначения на разъеме: `J7-1 = 2`, `J7-2 = 1`, `J7-3 = G`, `J7-4 = V`, `J7-5 = R`, `J7-6 = F`, `J7-7 = L`, `J7-8 = P`.
   - Группы `Q`: `Q1/Q2`, `Q3/Q4`, `Q5/Q6/Q7`, `Q8/Q9`, `Q10/Q11`. По виду больше похожи на светодиоды, чем на приемники; тип пока не утверждать.
   - Резисторы групп объединены на `V`, вторым концом идут в узел своей группы `Qx`.
   - `D13/D14/D15/D16/D17`: катоды приходят в те же узлы групп, где резистор соединён с группой `Qx`.
   - `R -> D13 -> Q2/Q1`.
   - `R -> D14 -> Q3/Q4`.
   - `F -> D15 -> Q5/Q6/Q7`.
   - `L -> D16 -> Q8/Q9`.
   - `L -> D17 -> Q10/Q11`.
   - `J7-1` с лейблом `2` идет на `D6`.
   - Дальше последовательно: `D6 -> D5 -> D4 -> D3 -> D2 -> D1`.
   - После `D1` дорожка возвращается на `J7-8 P`.
   - `J7-2` с лейблом `1` идет на `D11`.
   - Дальше последовательно: `D11 -> D10 -> D9 -> D8 -> D7`.
   - После `D7` дорожка тоже возвращается на `J7-8 P`.
   - Физический порядок пар по плате: `Q2-D1`, `D2-Q1`, `Q3-D3`, `D4-Q4`, `Q7-D5`, `D6-Q6`, `Q5-D7`, `D8-Q9`, `Q8-D9`, `D10-Q10`, `Q11-D11`.
   - `J7-4 V` идет на `TP25` и `D7`; `D7` идет на ключ `Q26`.
   - `Q26` питается от `3V3`, вероятно P-channel/high-side ключ.
   - Гейт `Q26` идет на `Q27`.
   - Управление `Q27` через сеть резисторов приходит на `PE7`.
2. `firmware-ready` - `J20`: одометрия переднего ролика (чёрно-белый диск). 2=приёмник->делитель->U10 ch1 (Шмитт), 4=эмиттер через 3.9R от 5V, 1/3=GND. U10:1->`PD2`. Драйвер `caster_odo.c` (EXTI2 счётчик), консоль `odo`.
3. `mapped` - два передних IR-приемника прямо на основной плате: OUT передний-левый -> `PB11`, передний-правый -> `PE15` (оба цифровые). Вместе с левым/правым/задним = 5 приёмников базы для направления на док.
4. `firmware-ready` - `J17`: левая боковая сборка. pin1=коммут.5V (датчики+мотор), pin2=привод эмиттеров->Q22, pin3=GND, pin4=cliff sense->`PC1`(ADC IN11), pin5=боковой трансивер RX->`PC0`(ADC IN10), pin6=база IR RX->`PD13`, pin7=удар бампера->`PB5`, pin8=прямые 5V. Cliff+side заведены в `.ioc` и читаются 8-канальным сканом `front_ir_bumper.c`.
5. `todo` - `J2`: правая боковая сборка: IR датчик, оптический концевик бампера, IR приемник.
6. `partial` - `J15`: задний IR приемник. OUT: `J15:3` через резистор -> `PD4` (цифровой вход). Осталось: VCC/GND (`J15:1`/`J15:2`).
7. `firmware-ready (sense)` - `J3`: правый передний cliff IR. sense `J3:3`->`PC4`(ADC IN14), живёт в 8-канальном скане.
8. `firmware-ready (sense)` - `J4`: левый передний cliff IR. sense `J4:3`->`PC2`(ADC IN12), живёт в 8-канальном скане.
9. `todo` - `J12`: сенсорная кнопка с RGB подсветкой.
10. `todo` - бузер.
11. `todo` - батарея: напряжение, ток, узел `R72/R73`, `U4`.
12. `todo` - вход зарядки / база: напряжение входа и детект подключения.
